Image Sampling & Quantization

A digital image is produced by sampling (digitizing spatial coordinates) and quantization (digitizing amplitude/intensity values). These three core formulas appear in virtually every numerical.

Core Formulas
Number of gray levels: L = 2k   (k = bit depth)
Number of pixels (resolution): M × N   (rows × columns)
Total storage (bits): b = M × N × k
For square images: b = N² × k
📐 Worked Numerical 1 Storage Size

Q: How many bits are required to store a 1024 × 1024, 8-bit grayscale image?

Step 1: M = N = 1024, k = 8 bits (so L = 2⁸ = 256 gray levels)
Step 2: b = N² × k = 1024² × 8 = 1,048,576 × 8 = 8,388,608 bits
Step 3: In bytes: 8,388,608 ÷ 8 = 1,048,576 bytes = 1 MB
✓
Formula from Eq. (2-12): b = M × N × k
📐 Worked Numerical 2 Bit Depth

Q: An image has 512 × 512 pixels and 64 gray levels. Find the bit depth and storage size.

Step 1: L = 64 = 2k → k = 6 bits
Step 2: b = 512 × 512 × 6 = 262,144 × 6 = 1,572,864 bits = 196,608 bytes ≈ 192 KB
⚠
Always check: L must be a power of 2. Gray levels = 2, 4, 8, 16, 32, 64, 128, 256 correspond to k = 1, 2, 3, 4, 5, 6, 7, 8.
📐 Worked Numerical 3 Resolution & Megabytes

Q: A 4000 × 3000 pixel, 24-bit color image has 3 channels (RGB), each 8 bits. Find total storage in MB.

Step 1: Bits per pixel = 3 × 8 = 24 bits
Step 2: Total bits = 4000 × 3000 × 24 = 288,000,000 bits
Step 3: In MB = 288,000,000 ÷ (8 × 10⁶) = 36 MB

Basic Relationships Between Pixels – Distance Measures

For pixels p at (x, y) and q at (u, v), three distance metrics are defined. These appear as direct formula-substitution problems in exams.

Euclidean: De(p,q) = √[(x−u)² + (y−v)²]    Eq. (2-19)
City-block: D₄(p,q) = |x−u| + |y−v|    Eq. (2-20)
Chessboard: D₈(p,q) = max(|x−u|, |y−v|)    Eq. (2-21)
📐 Worked Numerical 4 All Three Distances

Q: Find De, D₄, D₈ between pixel p at (1, 1) and pixel q at (4, 5).

De = √[(4−1)² + (5−1)²] = √[9 + 16] = √25 = 5
D₄ = |4−1| + |5−1| = 3 + 4 = 7
D₈ = max(|4−1|, |5−1|) = max(3, 4) = 4
💡
Key property: D₈ ≤ De ≤ D₄ always holds for integer-coordinate pixels. D₈ is always the smallest (or equal).

📐 Worked Numerical 5 m-adjacency / Dm distance

Q (Textbook pg. 82–83): Given pixels p, p₂, p₄ with value 1, and p₁, p₃ with value 0 or 1, find the Dm (shortest m-path) distance between p and p₄.

Case 1: p₁ = 0, p₃ = 0 → Dm(p, p₄) = 2 (direct m-path via p₂)
Case 2: p₁ = 1, p₃ = 0 → p and p₂ are no longer m-adjacent → path = p→p₁→p₂→p₄ → Dm = 3
Case 3: p₁ = 0, p₃ = 1 → path = p→p₂→p₃→p₄ → Dm = 3
Case 4: p₁ = 1, p₃ = 1 → path = p→p₁→p₂→p₃→p₄ → Dm = 4
⚠
Unlike D₄ and D₈, Dm depends on pixel values along the path, not just coordinates.

Gray-Level Transformations

Image Negative: s = (L − 1) − r    [for 8-bit: s = 255 − r]
Log transform: s = c · log(1 + r)    Eq. (3-4)
Power-law (Gamma): s = c · rγ    Eq. (3-5)
[γ < 1 → brightens dark images; γ > 1 → darkens bright images]
📐 Worked Numerical 6 Image Negative

Q: An 8-bit image (L = 256) has pixel values: 0, 50, 100, 200, 255. Find the negative image pixel values.

Formula: s = 255 − r
r = 0 → s = 255 − 0 = 255
r = 50 → s = 255 − 50 = 205
r = 100 → s = 255 − 100 = 155
r = 200 → s = 255 − 200 = 55
r = 255 → s = 255 − 255 = 0
💡
The negative of an image reverses light and dark areas — useful in medical imaging (e.g., mammograms).
📐 Worked Numerical 7 Log Transformation

Q: Apply the log transform s = c·log(1 + r) with c = 1 to pixel values r = 0, 10, 100, 1000 (for a high dynamic range image).

r = 0 → s = log(1 + 0) = log(1) = 0
r = 10 → s = log(1 + 10) = log(11) ≈ 2.40
r = 100 → s = log(1 + 100) = log(101) ≈ 4.62
r = 1000 → s = log(1 + 1000) = log(1001) ≈ 6.91

The dynamic range is compressed from 0–1000 to 0–6.91. This is used for Fourier spectrum display.

📐 Worked Numerical 8 Power-Law / Gamma Correction (Example 3.1)

Q: A CRT monitor has gamma γ = 2.5. A pixel value r = 0.5 (normalized) appears on the screen. What is the displayed value, and what gamma correction must be applied?

Monitor output: s = r2.5 = 0.52.5 = 0.5² × 0.50.5 = 0.25 × 0.707 ≈ 0.177 (image appears dark)
Correction exponent: Apply s = r1/2.5 = r0.4 as pre-processing
After correction on monitor: (r0.4)2.5 = r1.0 = 0.5 ✓ (faithful reproduction)
✓
Gamma correction formula: apply s = r(1/γ) before displaying on device with gamma γ.

Textbook rule (Eq. 3-5): γ < 1 (e.g., 0.4) → expands dark regions (brightens). γ > 1 (e.g., 3.0) → compresses bright regions (darkens washed-out images).

📐 Worked Numerical 9 Gamma on Integer Pixel Values

Q: Apply power-law transform with c = 1, γ = 0.5 to 8-bit pixel values r = 64, 128, 192. (L = 256)

Normalize to [0,1]: divide by 255. Apply transform. Rescale to [0,255].

r = 64: r_norm = 64/255 ≈ 0.251 → s_norm = 0.2510.5 ≈ 0.501 → s = 0.501 × 255 ≈ 128
r = 128: r_norm = 128/255 ≈ 0.502 → s_norm = 0.5020.5 ≈ 0.708 → s = 0.708 × 255 ≈ 181
r = 192: r_norm = 192/255 ≈ 0.753 → s_norm = 0.7530.5 ≈ 0.868 → s = 0.868 × 255 ≈ 221
💡
γ = 0.5 < 1 brightens the image — dark pixels are mapped to much higher values.

Histogram Equalization & Specification

The most common exam numerical. Always involves computing the PDF, CDF, and mapped intensity values.

PDF (Histogram): p(rk) = nk / MN    Eq. (3-14)
CDF → Equalized value: sk = (L−1) · Σ p(rj) for j=0 to k    Eq. (3-15)

📐 Worked Numerical 12 Example 3.4 — Continuous PDF & CDF

Q (Textbook Example 3.4, pg. 137): An image has intensity PDF: pr(r) = 2r/(L−1)² for 0 ≤ r ≤ L−1. Show the equalization transformation and verify the output PDF is uniform.

Transformation (Eq. 3-11): s = T(r) = (L−1) ∫₀ʳ pr(w)dw
= (L−1) × ∫₀ʳ [2w/(L−1)²] dw = (L−1) × [w²/(L−1)²]₀ʳ = r²/(L−1)
Verify output is uniform: ps(s) = pr(r)|dr/ds|
Since s = r²/(L−1), so ds/dr = 2r/(L−1), then |dr/ds| = (L−1)/(2r)
ps(s) = [2r/(L−1)²] × [(L−1)/(2r)] = 1/(L−1) ✓ (uniform)

Spatial Filtering – Smoothing & Sharpening

Apply a kernel/mask over a pixel's neighborhood and compute the weighted sum. The center of the kernel is placed on the pixel of interest.

Spatial Convolution: g(x,y) = Σ Σ w(s,t) · f(x+s, y+t)
Averaging (Box) filter kernel (3×3): all 9 elements = 1/9
Normalized kernel: coefficients sum = 1 (preserves average brightness)

Q (Textbook pg. 175): A 3×3 neighborhood has pixel values: 10, 20, 20, 20, 15, 20, 20, 25, 100. Find the median-filtered output value.

2D Discrete Fourier Transform (DFT) & Inverse

1D DFT: F(u) = Σ f(x) · e−j2πux/M for x=0..M−1   Eq. (4-44)
1D IDFT: f(x) = (1/M) Σ F(u) · ej2πux/M for u=0..M−1

2D DFT: F(u,v) = ΣΣ f(x,y) · e−j2π(ux/M + vy/N) Eq. (2-61)
2D IDFT: f(x,y) = (1/MN) ΣΣ F(u,v) · ej2π(ux/M + vy/N) Eq. (2-62)

Euler's formula: e−jθ = cos(θ) − j·sin(θ)
📐 Worked Numerical 18 Example 4.4 — 1D DFT (4-point sequence)

Q (Textbook Example 4.4, pg. 229): Compute the DFT of the 4-sample sequence f = {1, 2, 4, 4} (M = 4).

F(u) = Σx=03 f(x) · e−j2πux/4
F(0): = Σ f(x) · e⁰ = f(0)+f(1)+f(2)+f(3) = 1+2+4+4 = 11
F(1): = 1·e⁰ + 2·e−jπ/2 + 4·e−jπ + 4·e−j3π/2
   = 1·(1) + 2·(−j) + 4·(−1) + 4·(j)
   = 1 − 2j − 4 + 4j = −3 + 2j
F(2): = 1·e⁰ + 2·e−jπ + 4·e−j2π + 4·e−j3π
   = 1 + 2·(−1) + 4·(1) + 4·(−1) = 1 − 2 + 4 − 4 = −1
F(3): = conjugate of F(1) = −3 − 2j (for real input)

Verify IDFT – recover f(0):

f(0) = (1/4)·[F(0)+F(1)+F(2)+F(3)] = (1/4)·[11+(−3+2j)+(−1)+(−3−2j)] = (1/4)·[4] = 1 ✓
💡
For real-valued input, F(M−u) = F*(u) (complex conjugate symmetry). F(0) is always real and equals the sum of all samples.

Frequency Domain Filtering

Steps for frequency-domain filtering:
1. Pad f(x,y) to size P×Q (to avoid wrap-around error)
2. Compute F(u,v) = DFT[f(x,y)]
3. Multiply: G(u,v) = H(u,v) · F(u,v) [H = filter transfer function]
4. Inverse DFT: g(x,y) = IDFT[G(u,v)]
5. Extract top-left M×N portion

Computational advantage of FFT vs non-separable spatial: Cn(m) = m² / (4·log₂M) Eq. (4-1)
vs separable spatial kernel: Cs(m) = m / (2·log₂M) Eq. (4-2)
📐 Worked Numerical 20 FFT Computational Advantage

Q (Textbook Eq. 4-1, pg. 205): For a 512×512 image (M = 512) and a 15×15 non-separable kernel, is FFT or spatial convolution faster?

Cn(m) = m² / (4·log₂M) = 15² / (4·log₂512) = 225 / (4×9) = 225/36 = 6.25
Since Cn(15) = 6.25 > 1, the FFT approach is 6.25× faster than non-separable convolution.
✓
For large kernels (m > ~3 for typical image sizes), frequency domain filtering is computationally superior.
📐 Worked Numerical 21 Ideal Lowpass Filter (ILPF) Cutoff

Q: For an ideal lowpass filter in the frequency domain, define the transfer function. Given D₀ = 50, is a frequency component at (u,v) = (30, 40) passed or rejected?

H(u,v) = 1 if D(u,v) ≤ D₀
H(u,v) = 0 if D(u,v) > D₀
where D(u,v) = √(u² + v²) (distance from origin)
D(30, 40) = √(30² + 40²) = √(900 + 1600) = √2500 = 50
Since D(u,v) = D₀ = 50, this frequency is exactly at the cutoff → passed (H = 1)
⚠
Low frequencies (near origin) → slowly varying regions (smooth backgrounds). High frequencies → edges and fine details.
📐 Worked Numerical 22 Highpass from Lowpass

Q: If a lowpass filter has HLP(u,v), what is the transfer function of the corresponding highpass filter?

HHP(u,v) = 1 − HLP(u,v)
For ILPF with D₀ = 50, if HLP(u,v) = 1 at D ≤ 50, then HHP(u,v) = 0 at D ≤ 50 (blocks low freq)
And HHP(u,v) = 1 at D > 50 (passes high freq = edges)
💡
In spatial domain: highpass kernel = unit impulse − lowpass kernel. Unsharp masking is exactly this operation.
📐 Worked Numerical 23 Example 4.15 — Spatial Kernel ↔ Frequency Transfer Function

Concept (Textbook Example 4.15): A spatial kernel can be embedded in a zero-padded array and DFT-computed to obtain its frequency-domain equivalent H(u,v). This allows comparison of spatial vs frequency filtering.

Key insight: Sobel kernel (from Section 6) embedded in a P×Q zero-padded array → compute 2D DFT → get HSobel(u,v)
The kernel has purely imaginary DFT (since it is an odd function) → results are identical to spatial convolution.
✓
Spatial domain convolution ≡ multiplication in the frequency domain (Convolution theorem).
